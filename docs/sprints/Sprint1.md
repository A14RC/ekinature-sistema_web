Actividades Realizadas:

Inicialización del Repositorio: Configuración del control de versiones en GitHub con la estructura de ramas (main / develop) y definición del .gitignore para entornos Node.js.

Arquitectura del Sistema: Se definió una estructura de carpetas basada en el patrón Modelo-Vista-Controlador (MVC), separando claramente el Backend (API REST en Express) del Frontend (Cliente React).

Diseño de Base de Datos: Elaboración del diagrama Entidad-Relación (DER) preliminar. Se normalizó la lógica de negocio separando la entidad Pedidos de la entidad Entregas para permitir la reprogramación de envíos sin duplicidad de datos transaccionales.

Planificación: Desglose de requerimientos funcionales en Historias de Usuario cargadas en GitHub Projects.
